앞선 MLP모델의 오차 행렬 분석을 통해 모델이 학습을 한 것이 아닌 단순히 불량품이라고만 하는 치명적인 결점이 발견되었습니다.
MLP를 사용하더라도 분석이 가능할거라고 생각했던 착오입니다.
금속 부품의 흠집이나 파임 같은 2차원적인 정보를 1차원으로 펼치면서 ('Flatten') 잃어버려서 판단이 불가능하기 때문입니다.
이를 해결하기위해 이미지의 공간정보를 그대로 유지하며 특징을 추출하는 CNN모델을 사용했습니다.

# 배운 내용

코드 연습을 위한 프로젝트였지만, 단순 연습을 넘어 실무적인 관점에서 도움이 되었습니다.

 1. 정확도의 함정과 오차행렬의 중요성

    정확도 수치만 보았을 땐 어느정도 학습이 되었다고 생각했고, 조금씩 늘려가야겠다는 생각을 했습니다.
    그러나 오차 행렬을 직접 뜯어본 결과, 모델이 모든 예측을 단일 클래스(불량품)로 쏠리게 판단하는 심각한 오류를 범하고 있음을 짚어냈습니다.
     **단일 지표에 의존하지 않고, 모델의 실제 판단 근거를 교차 검증하는 심화 데이터 분석의 중요성**을 배웠습니다.

 2. 제조 현장에서의 FN(False Negative)과 FP(False Positive)
CNN 모델의 오차 행렬에서 단 2장의 오답이 발생했는데, 이 2장은 **FN(실제 불량인데 정상으로 판별)**이었습니다.
제조업 비전 검사에서는 멀쩡한 부품을 불량으로 오해해 버리는 것(FP)보다, **불량품을 정상으로 오해하여 고객에게 출하하는 것(FN)**이 훨씬 치명적인 리스크입니다. 
99.72%라는 높은 정확도에 안주하지 않고, 현업의 관점에서는 이 2장의 Miss를 잡기 위한 추가적인 데이터 보완이 필요하다는 통찰을 얻었습니다.
    **약점 집중 공략 (Hard Example Mining)**
    **판단 기준선 높게 잡기 (Threshold 조절)**
    **불량 놓치면 벌점 크게 주기 (Weighted Loss)**
이러한 방식들을 사용한다면 더 좋은 결과가 있을것이라고 생각합니다.

 3. 왜 컬러(RGB) 대신 흑백(Grayscale)을 사용했는가?
금속 펌프 부품의 표면 결함은 색상 정보보다 **명암(그림자의 깊이)**으로 훨씬 명확하게 드러납니다.
 따라서 3채널(RGB) 데이터를 1채널(Grayscale)로 변환하여 불필요한 연산량을 줄이고, 모델이 결함의 '형태'에만 집중할 수 있도록 유도했습니다.
